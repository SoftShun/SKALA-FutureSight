# 미래 기술 트렌드 분석 미니 프로젝트 PRD (Product Requirements Document)

버전: 1.0
작성일: 2025-05-20
작성자: 안시현

## 1. 제품 개요

### 1.1 목적 및 비전
미래 기술 트렌드 분석 도구는 사용자가 선택한 기술 분야의 최신 연구 논문, 기술 뉴스, 시장 동향 등을 종합적으로 분석하여 향후 5년 내(2025-2030) 기술 트렌드를 예측하는 보고서를 생성합니다. 이를 통해 기업의 전략 기획자, 연구 개발자, 투자자, 기술 관련 의사결정자들이 전략적 R&D 투자 방향 설정, 신사업 기회 발굴, 기술 로드맵 수립에 활용할 수 있습니다.

### 1.2 목표 사용자
- 기업의 전략 기획자
- 연구 개발자
- 투자자
- 기술 관련 의사결정자

### 1.3 주요 기능 요약
- 다양한 기술 분야(AI, 로보틱스, 에너지 기술, 생명과학/바이오테크) 중 관심 분야 선택
- 선택 분야 내 세부 분야 선택 가능 (예: AI → 생성형 AI)
- 선택 분야에 대한 자동 데이터 수집 및 분석
- 분야 간 융합 가능성 및 시너지 분석
- 다양한 형식(MD, PDF, DOCS)의 분석 보고서 생성
- 다국어(한국어, 영어) 보고서 지원

## 2. 제품 요구사항

### 2.1 기능적 요구사항

#### 2.1.1 사용자 입력 및 설정
- 사용자는 관심 있는 기술 분야를 1-3개 선택할 수 있어야 함
- 선택한 분야 내에서 세부 분야 선택 가능
- 보고서 언어(한국어/영어) 선택 가능
- 보고서 형식(MD/PDF/DOCS) 선택 가능
- 분석 깊이 설정 가능 (RAG 적용 문서 선택)

#### 2.1.2 데이터 수집 및 분석
- 선택된 분야에 대한 최신 연구 논문, 특허 정보, 기술 뉴스, 시장 보고서 데이터 수집
- 다음 기준에 따른 트렌드 예측:
  - 논문 발행 빈도 및 인용 수
  - 특허 출원 동향
  - 벤처 투자 동향
  - 기술 뉴스 빈도
  - 기업 R&D 투자 동향
- 분야별 주요 트렌드 분석
- 분야 간 융합 가능성 및 시너지 분석
- 분야별 Top 5 유망 기술 선정

#### 2.1.3 보고서 생성
- 정해진 보고서 구조에 맞게 마크다운 형식의 보고서 생성
- 보고서 상단에 전체 내용을 요약한 'SUMMARY' 섹션 포함
- 분야별 주요 트렌드 요약
- 분야 간 융합 가능성과 시너지 분석
- 향후 5년 내 주목해야 할 Top 기술 리스트
- 기업 대응 전략 제언
- 생성된 마크다운을 PDF 또는 DOCS 형식으로 변환 기능

#### 2.1.4 다국어 지원
- 한국어, 영어로 보고서 생성 가능

### 2.2 비기능적 요구사항

#### 2.2.1 성능
- 보고서 생성 시간은 합리적인 수준으로 유지 (최대 5분 이내)
- 웹 검색 API 호출 횟수 최적화

#### 2.2.2 보안
- API 키 및 민감한 정보는 환경 변수를 통해 관리
- 생성된 보고서는 로컬에만 저장

#### 2.2.3 확장성
- 추가 언어 지원 가능한 구조 설계
- 새로운 기술 분야 추가 용이한 구조 설계

#### 2.2.4 유지보수성
- 모듈화된 코드 구조
- 적절한 문서화 및 주석
- 테스트 코드 작성

## 3. 기술 스택

### 3.1 개발 환경
- Python 3.12
- venv 가상환경

### 3.2 주요 라이브러리 및 프레임워크
- LangChain / LangGraph: 에이전트 구현 및 상태 관리
- ChromaDB: 벡터 저장소
- PyPDF2/ReportLab: PDF 변환
- python-docx: DOCS 변환
- tiktoken: 토큰 카운팅
- typer: CLI 인터페이스

### 3.3 외부 API 및 서비스
- OpenAI API: LLM 사용
- Brave Search API: 웹 검색

## 4. 시스템 아키텍처

### 4.1 시스템 구성 요소
- 사용자 인터페이스: CLI를 통한 입력 수집
- Coordinator Agent: 전체 프로세스 관리
- Research Agent: 데이터 수집 및 분석
- Report Agent: 보고서 생성 및 형식 변환
- 벡터 저장소: 수집된 문서의 임베딩 저장

### 4.2 데이터 흐름
1. 사용자 입력 → 분야 선택
2. 선택된 분야 → Coordinator Agent → 작업 계획 수립
3. Coordinator Agent → Research Agent → 데이터 수집 및 분석
4. Research Agent → Coordinator Agent → 분석 결과 전달
5. Coordinator Agent → Report Agent → 보고서 생성
6. Report Agent → 형식 변환 → 최종 보고서 사용자에게 제공

## 5. 개발 일정 및 우선순위

### 5.1 핵심 기능(MVP)
1. 기본 프로젝트 구조 설정
2. 사용자 인터페이스 구현
3. Brave Search 통합
4. OpenAI API 통합
5. LangGraph 기반 에이전트 구현
6. 마크다운 보고서 생성
7. 한국어/영어 지원
